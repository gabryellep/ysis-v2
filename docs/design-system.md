# Ysis V2 Design System

Status: foundation only. This document defines the base visual system; it does not define the final landing layout.

## Product Feeling

Ysis V2 must feel premium, warm, private, cinematic and human. It must not look like a medical dashboard, generic SaaS, marketplace, menstrual-cycle app, hospital form, chatbot shell or the old Ysis.

## Color Tokens

- `--color-canvas`: off-white base.
- `--color-paper`: warm white surface.
- `--color-rose`: burned rose for emotional warmth.
- `--color-rose-soft`: softer rose for glow and highlights.
- `--color-lavender`: lavender for care and calm technology.
- `--color-lavender-deep`: deeper lavender for expressive text.
- `--color-wine`: soft wine for primary actions and anchors.
- `--color-ember`: warm accent.
- `--color-honey`: subtle warm light.
- `--color-ink`: readable warm dark text.
- `--color-muted`: quiet support text.

## Surface Tokens

- `--surface-glass`: translucent warm surface.
- `--surface-glass-strong`: stronger translucent surface.
- `--stroke-glass`: subtle border for privacy-first layers.
- `--blur-glass`: shared backdrop blur.

## Depth Tokens

- `--shadow-veil`: soft elevated surface shadow.
- `--shadow-glow`: rose glow for primary focus.
- `--shadow-bloom`: lavender bloom for floating mockups and cinematic depth.

## Radius Tokens

- `--radius-soft`: compact controls.
- `--radius-silk`: premium translucent panels.
- `--radius-orbit`: pills and soft action controls.

## Typography Tokens

- `--font-sans`: interface text.
- `--font-display`: expressive headings.

Use large expressive type only where it improves the composition. Compact panels must use quieter sizes.

## Component Principles

- Buttons should feel tactile and warm, not SaaS-flat.
- Cards must be translucent surfaces for meaningful content, not a generic grid.
- Mockups should float and show product direction without becoming the final landing.
- Safety notices must be visible, calm and non-diagnostic.
